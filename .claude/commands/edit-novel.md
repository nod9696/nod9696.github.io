神の降る街の商業レベル編集チームを起動してください。

## チームメンバー
- 🏗️ 構造編集者 — プロット構造・ペーシング・フック
- ✍️ 文章担当編集者 — 文体・リズム・固有用語
- 👤 キャラクター編集者 — キャラの一貫性・魅力・感情移入
- 🌍 世界観編集者 — ロア整合性・雰囲気・術語の正確性
- 💼 商業担当編集者 — 市場評価・商業可能性・差別化
- 📋 チーフ編集者 — 総合評価・優先改善事項の統合
- ✍️ リライター — 改善指示に基づく修正稿生成

## 実行手順

1. 原稿テキストを取得する：
   - `$ARGUMENTS` にテキストが含まれていればそれを使用
   - IDEで選択中のテキストがあればそれを使用
   - どちらもない場合はユーザーに「編集したい原稿を貼り付けてください」と尋ねる

2. 以下の世界観を各エージェントのプロンプトに必ず含める：

```
【作品「神の降る街」の世界観】
- 迷宮都市ナインフォール：神の遺骸が周期的に降る都市
- 神の遺骸：空から降る神の一部。魔力の塊。帝国が独占するエネルギー資源
- 神害：遺骸の影響で発生する災害や怪物
- 神憑き：遺骸が寄生した者。力を得るが短命
- ウタ（神の声）：死体から発され、人間をマヨイへ誘う声
- マヨイ：異形に変化した元人間
- 緘葬：神憑きの死体を葬る特殊な儀式
- 主人公カナタ・イグル：右腕に遺骸が寄生した神憑き。寡黙・無表情。忌み名「マヨワズ」
- ヒロイン（仮：リリス）：神の声に惹かれる少女。敵か味方か曖昧
- 帝国：ルキウス（宰相）、グリーフ（騎士）、エレオノーラ（科学者）
- ジャンル：ダークファンタジー×スチームパンク×ライトノベル
- 文体方針：退廃・神秘・儀式的語彙を多用。テンポはLN読者向けに軽妙
```

3. Agentツールで5人の編集者を**並行**（同一メッセージで複数のAgentツール呼び出し）起動する：

   **構造編集者**のプロンプト：
   「あなたはLN構造編集の専門家です。[世界観]。以下の原稿を分析：①プロット構造・起承転結 ②テンポとペーシング ③冒頭フックの強度 ④情報開示のバランス ⑤場面転換の論理性。評価点(/5)・問題箇所(原文引用付き)・改善提案(優先度順)を出力。原稿：[原稿]」

   **文章担当編集者**のプロンプト：
   「あなたはLN文体・校正の専門家です。[世界観]。以下の原稿を分析：①文体の一貫性と退廃・神秘トーンの維持 ②文章リズム・読みやすさ ③固有用語の効果的使用 ④描写バランス ⑤セリフの質。評価点(/5)・問題箇所(引用+修正例)・文体強化提案を出力。原稿：[原稿]」

   **キャラクター編集者**のプロンプト：
   「あなたはLNキャラクター専門編集者です。[世界観]。以下の原稿を分析：①カナタの寡黙・無表情・感情抑制の表現精度 ②リリスのミスリード要素の維持 ③各キャラのセリフ・行動の個性 ④キャラ間の緊張感 ⑤感情移入の仕掛け。評価点(/5)・キャラ別詳細評価・改善案を出力。原稿：[原稿]」

   **世界観編集者**のプロンプト：
   「あなたはファンタジー世界観チェック専門編集者です。[世界観]。以下の原稿を分析：①固有用語の一貫性と正確性 ②スチームパンク×ダークファンタジーの融合度 ③世界観ルールの矛盾・整合性 ④退廃・神秘的空気感の持続 ⑤新規読者への自然な説明。評価点(/5)・矛盾指摘(引用付き)・没入感向上提案を出力。原稿：[原稿]」

   **商業担当編集者**のプロンプト：
   「あなたはLN商業出版・市場評価専門編集者です。[世界観]。以下の原稿を評価：①現在のLN市場トレンドとの整合性 ②ターゲット読者層への訴求力 ③類似作品との差別化 ④26話シリーズとしての商業継続力 ⑤一次選考通過可能性。商業可能性評価(/5)・強みと致命的弱点・最優先改善事項TOP3を出力。原稿：[原稿]」

4. 全エージェントの完了を待ち、結果を受け取る

5. チーフ編集者として5つのフィードバックを統合し、以下の形式で総合評価を出力：

```
## 📊 総合評価
商業レベル到達度：XX%
現状の位置づけ：（一言で）

## 🎯 最優先改善事項（TOP3）
1. **[タイトル]** — 詳細説明
2. **[タイトル]** — 詳細説明
3. **[タイトル]** — 詳細説明

## ✨ 強みとして活かすべき点
## 📝 次稿への具体的指示
```

6. チーフ編集者の指示に基づき、リライターとして修正稿を生成する：
   - 固有用語（ウタ、緘葬、マヨイ等）は原文のまま使用
   - カナタの寡黙・無表情を崩さない
   - 退廃・神秘的雰囲気を強化
   - 原文の長さ±30%以内
   - リライト稿のみ出力（解説不要）

## 注意事項
- 外部APIキー・外部スクリプト不要。Claude Code内で完結
- 5人の編集者は必ず並行起動（同一メッセージで複数Agent呼び出し）
- 原稿は1000〜3000文字程度が最適
