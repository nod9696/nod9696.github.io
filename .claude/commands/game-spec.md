神の降る街のノベルADVゲーム仕様書作成チームを起動してください。

## チームメンバー
- 📖 シナリオ仕様担当 — チャプター構成・選択肢・分岐フロー・エンディング条件
- ⚙️ システム仕様担当 — ゲームエンジン・セーブ/ロード・フラグ管理・画面遷移
- 👥 キャラクター仕様担当 — スプライト差分・ボイス仕様・好感度パラメータ
- 🎨 ビジュアル仕様担当 — 背景/CG一覧・UI/UX・演出エフェクト・カラーパレット
- 🎵 サウンド仕様担当 — BGM・SE・ボイス収録・音楽的世界観
- 📋 統括ディレクター — 矛盾チェック・MVP優先度・依存関係整理・最終統合

## 世界観リファレンス
```
【作品「神の降る街」ゲーム概要】
ジャンル: ノベルADV（アドベンチャーゲーム）
- 迷宮都市ナインフォール：神の遺骸が周期的に降る都市
- 神の遺骸：空から降る神の一部。魔力の塊。帝国が独占するエネルギー資源
- 神害：遺骸の影響で発生する災害や怪物
- 神憑き：遺骸が寄生した者。力を得るが短命
- ウタ（神の声）：死体から発され、人間をマヨイへ誘う声
- マヨイ：異形に変化した元人間
- 緘葬（かんそう）：神憑きの死体を葬る特殊な儀式
- 主人公カナタ・イグル：右腕に遺骸が寄生した神憑き。寡黙・無表情。忌み名「マヨワズ」
- ヒロイン（仮：リリス）：神の声に惹かれる少女。敵か味方か曖昧
- ジャンル：ダークファンタジー×スチームパンク×ライトノベル
- ターゲット：10代後半〜20代前半。プレイ時間：全ルート約20〜30時間
- 分岐構造：マルチルート・マルチエンディング（グッドエンド/バッドエンド/トゥルーエンド）
```

## 実行手順

テーマを取得する：
- `$ARGUMENTS` に内容が含まれていればそれを使用
- なければユーザーに「仕様書のテーマを教えてください」と尋ねる

**5人の担当者を Agent ツールで並列起動する（必ず同一メッセージで全員同時に起動すること）：**

### 📖 シナリオ仕様担当 への指示
```
あなたは「神の降る街」ノベルADVのシナリオ設計専門家です。
テーマ「{THEME}」について、以下の仕様書セクションをMarkdownで作成してください：

1. シナリオ構造（チャプター/シーン分割）
2. 選択肢とフラグ管理（分岐条件を明示。柔軟な分岐を最優先で設計）
3. ルート分岐図（ASCIIフロー図）
4. キーシーン一覧（タイトル・概要・感情的目標）
5. エンディング条件と種類（グッド/バッド/トゥルー）
6. テキスト量の目安

演出の自由度を最大にするため、フラグは細粒度で設計し、後からシーンを追加・差し込みやすい構造にすること。
```

### ⚙️ システム仕様担当 への指示
```
あなたは「神の降る街」ノベルADVのシステム設計専門家です。
テーマ「{THEME}」について、以下の仕様書セクションをMarkdownで作成してください：

1. 推奨ゲームエンジン（Ren'Py / Unity + Novel Engine / Godot 等から選定・理由付き）
2. コアシステム仕様（セーブ/ロード・既読スキップ・オートモード・バックログ・早送り）
3. フラグ・変数管理システム（JSON構造案・グローバル変数/ローカル変数の設計）
4. 画面遷移仕様（状態機械図）
5. 設定メニュー仕様
6. 演出コマンド仕様（スクリプト言語レベルで、どんな演出命令が必要か）

「柔軟な分岐」と「最大限の演出表現」を実現するための技術選定と設計を最優先とすること。
```

### 👥 キャラクター仕様担当 への指示
```
あなたは「神の降る街」ノベルADVのキャラクター設計専門家です。
テーマ「{THEME}」について、以下の仕様書セクションをMarkdownで作成してください：

1. キャラクター一覧（登場シーン・役割・関係性）
2. スプライト差分リスト（表情×衣装×ポーズの組み合わせ表）
3. ボイス仕様（声質・口調・感情タグ・収録ディレクション）
4. キャラクター固有演出（登場/退場エフェクト案）
5. 好感度・フラグパラメータ仕様（存在する場合）
```

### 🎨 ビジュアル仕様担当 への指示
```
あなたは「神の降る街」ノベルADVのビジュアル設計専門家です。
テーマ「{THEME}」について、以下の仕様書セクションをMarkdownで作成してください：

1. 背景（BG）一覧（シーン名・時間帯・天候・バリエーション）
2. イベントCG一覧（シーン番号・内容・感情的役割・ルート条件）
3. UI/UX仕様（テキストボックス・メニュー・選択肢画面レイアウト）
4. 演出エフェクト仕様（画面効果・トランジション・シェイク・フラッシュ・フィルター）
5. カラーパレット方針（ダークファンタジー×スチームパンクに合わせたトーン設計）
6. 解像度・アスペクト比仕様

「最大限の演出表現」のため、エフェクト仕様は特に豊富に定義すること。
```

### 🎵 サウンド仕様担当 への指示
```
あなたは「神の降る街」ノベルADVのサウンド設計専門家です。
テーマ「{THEME}」について、以下の仕様書セクションをMarkdownで作成してください：

1. BGM一覧（トラック名・使用シーン・ムード・テンポ・楽器編成の方向性）
2. SE（効果音）一覧（種類・使用タイミング・音の質感）
3. ボイス収録仕様（感情タグ付けルール・優先度・ディレクション指示）
4. 環境音仕様（アンビエントサウンドの使用箇所）
5. 音楽的世界観定義（参考楽曲・推奨スタイル）
6. 実装仕様（ループポイント・フェード設定）
```

全員の出力が揃ったら、**統括ディレクターとして**以下を行う：

1. 各セクションの矛盾・不整合をチェックし修正
2. MVP必須 vs 後回し可能な要素を分類
3. 実装依存関係を整理（「Aが完了してからBに着手」）
4. 全セクションを統合した最終仕様書をMarkdownで出力

## 保存
最終仕様書を `f:/Claude/gamespec_{テーマ30文字}_{YYYYMMDD}.md` に Write ツールで保存し、パスを報告する。

## 注意事項
- Agent ツールで5人を **必ず並列（同一メッセージ内）** で起動する
- 各担当者の指示内の `{THEME}` は実際のテーマ文字列に置換すること
- 外部スクリプトや API は使用しない（Claude Code 内で完結）
