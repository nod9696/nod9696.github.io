ゲーム仕様書の指定セクションを差分更新してください。

## 概要
既存の仕様書セクションを、最新のソース資料と比較して差分更新する。
変更があった部分のみを更新し、変更ログをセクション末尾に追記する。

## 引数
`$ARGUMENTS` の形式：
- `{project} {section}` — 特定セクションを更新（例: `Delight characters`）
- `{project} all` — 全セクションを更新
- `{project}` のみ — セクション選択メニューを表示
- 空 — プロジェクト一覧を表示して選択を促す

有効なセクションID: `system_design`, `scenario`, `characters`, `visual`, `sound`, `flags`

## 手順

### 1. マニフェスト読み込み
`f:/Claude/specs/{project}/manifest.json` を Read ツールで読み込む。
存在しない場合は「先に `/spec-init {project}` を実行してください」と案内する。

### 2. ソース資料の変更検出
manifest.json の `sources` に記載された各ファイルについて：
- ファイルの存在確認（Glob）
- ファイルの最終更新日時を取得（`ls -l` で確認）
- manifest の `updated` 日時と比較
- **更新が必要なソース** の一覧を作成

更新検出結果をユーザーに報告：
```
📋 変更検出結果:
  ✅ character_sheet_polished_v2.md — 更新あり（2026-03-27 > 仕様書 2026-03-25）
  ⏸️ novel_common_v2.txt — 変更なし
  🆕 新規ファイル検出: relationship_analysis_detailed.md
```

### 3. 更新対象の決定
- 引数で `all` が指定された場合：変更のあったソースに依存する全セクションを更新
- 特定セクションが指定された場合：そのセクションのみ更新
- セクション未指定の場合：影響を受けるセクションを提示してユーザーに選択を促す

### 4. 差分更新の実行

更新対象セクションごとに、以下の手順で更新する。
**複数セクションの場合は Agent ツールで並列起動する。**

各エージェントへの指示テンプレート：
```
あなたはノベルADVゲーム仕様書の更新担当です。

【タスク】
「{セクション名}」セクションを、最新のソース資料に基づいて差分更新してください。

【現在の仕様書】
{現在のセクションファイルの内容}

【更新されたソース資料】
{変更のあったソース資料の内容}

【ルール】
1. 既存の内容をベースに、変更・追加が必要な箇所のみ更新する
2. 既存の構造・フォーマットを維持する
3. 削除された情報は「[削除] 理由」として残す（確認用）
4. 新規追加部分には「[NEW]」マーカーを付ける
5. 変更部分には「[UPDATED]」マーカーを付ける
6. セクション末尾に変更ログを追記：
   ```
   ## 変更ログ
   - {YYYY-MM-DD} v{N}: {変更概要}
   ```

結果をMarkdownで出力してください。Writeツールは使わず、テキストとして返してください。
```

### 5. 統合・保存
全エージェントの出力が揃ったら：

1. 各セクションファイルを Write ツールで上書き保存
2. フラグ・変数セクションが他セクションの変更に影響される場合、追加で整合性更新
3. manifest.json を更新：
   - `updated` を今日の日付に
   - 各セクションの `version` をインクリメント
   - 各セクションの `updated` を今日の日付に
   - 新規ソースファイルがあれば `sources` に追加

### 6. 変更マーカーのクリーンアップ確認
更新後、[NEW] と [UPDATED] マーカーの一覧をユーザーに提示：
```
📝 変更箇所一覧:
  [NEW] characters.md L45: キャラクター「ナホァ」の追加
  [UPDATED] characters.md L12: ライの深層心理セクション更新
  [UPDATED] flags.md L30: l_aff_nahua 変数追加

マーカーを削除して確定しますか？（y/n）
```

ユーザーが確認したら [NEW] [UPDATED] [削除] マーカーを除去して最終保存。

### 7. 完了報告
- 更新されたセクション一覧
- 変更点のサマリー
- 整合性に関する注意事項（あれば）
- 次のステップの提案
