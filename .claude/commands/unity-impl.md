## チームメンバー
- 💬 ダイアログエンジン担当 — シナリオデータ構造・テキスト表示・選択肢分岐
- 🏗️ シーン・アーキテクチャ担当 — シーン構成・GameManager・セーブ/ロード
- 🖼️ UI/UX担当 — TextMeshPro・選択肢UI・メニュー・バックログ
- ✨ ビジュアル・演出担当 — スプライト管理・エフェクト・ポストプロセス

- 🔧 テクニカルディレクター — 整合性チェック・実装優先度・依存関係・最終統合

## 実行手順

1. テーマを取得する：
   - `$ARGUMENTS` に内容が含まれていればそれを使用
   - どちらもない場合はユーザーに「実装したいシステム・機能を入力してください（例：テキスト表示システム、セーブ/ロード）」と尋ねる

2. 以下の世界観を各エージェントのプロンプトに必ず含める：

```
【作品「神の降る街」Unity実装コンテキスト】
- ジャンル: ノベルADV（Unityで実装）
- ターゲット: PC/スマートフォン (Unity 2022 LTS以降)
- UI解像度: 1920x1080基準、スケーラブル
- 迷宮都市ナインフォール：神の遺骸が周期的に降る都市
- 神害・神憑き・ウタ（神の声）・マヨイ・緘葬（かんそう）
- 主人公カナタ・イグル：右腕に遺骸が寄生した神憑き。寡黙・無表情
- ヒロイン（仮：リリス）：神の声に惹かれる少女
- アーキテクチャ方針：ADVエンジンはカスタム実装。シナリオデータは外部JSON。ScriptableObject活用。UniTask(async/await)。PersistentScene+ContentScene構成
```

3. Agentツールで5人の担当を**並行**（同一メッセージで複数のAgentツール呼び出し）起動する：

   **ダイアログエンジン担当**のプロンプト：
   「あなたはUnityでノベルADVのコアエンジンを実装する専門エンジニアです。[世界観]。以下のテーマについてUnity実装ガイドを作成してください：①シナリオデータ構造（JSON/ScriptableObject設計）②テキスト逐次表示システム（ルビ・特殊文字対応）③選択肢分岐システム（フラグ管理・条件分岐ロジック）④シナリオ進行管理クラス設計⑤キーボード/タップ入力によるテキスト送り。クラス設計図＋C#実装コード（コメント付き）＋UniTask/async対応設計を出力してください。テーマ：[テーマ]」

   **シーン・アーキテクチャ担当**のプロンプト：
   「あなたはUnityプロジェクトのアーキテクチャ設計専門エンジニアです。[世界観]。以下のテーマについてUnity実装ガイドを作成してください：①シーン構成設計（PersistentScene/ContentScene分離）②GameManager/SceneController設計③セーブ・ロードシステム（データ構造・永続化方法）④フラグ管理システム⑤シーン遷移・フェード演出。Assetsフォルダ構成案＋C#実装コード（コメント付き）を出力してください。テーマ：[テーマ]」

   **UI/UX担当**のプロンプト：
   「あなたはUnity UI（uGUI/TextMeshPro）専門エンジニアです。[世界観]。以下のテーマについてUnity実装ガイドを作成してください：①テキストボックスUI（TextMeshPro設定・フォント・ルビ対応）②選択肢UIシステム（動的生成・アニメーション）③メインメニュー・タイトル画面UI④バックログUI（スクロール・再生）⑤設定メニュー（音量・テキスト速度）。UI階層構造＋C#実装コード（DOTweenアニメーション込み）を出力してください。テーマ：[テーマ]」

   **ビジュアル・演出担当**のプロンプト：
   「あなたはUnityのビジュアル演出・URP専門エンジニアです。[世界観]。以下のテーマについてUnity実装ガイドを作成してください：①キャラクタースプライト管理（立ち絵表示・差分・フェード）②背景画像管理とトランジション③画面エフェクト（揺れ・フラッシュ・退廃的雰囲気演出）④神の遺骸/ウタ/緘葬などの特殊演出アイデアと実装⑤URP/ポストプロセス設定（vignette・color grading）。C#実装コード（UniTask対応）＋ShaderGraphスニペット（必要な場合）を出力してください。テーマ：[テーマ]」

   **オーディオ担当**のプロンプト：
   「あなたはUnityオーディオシステム専門エンジニアです。[世界観]。以下のテーマについてUnity実装ガイドを作成してください：①AudioManagerの設計（BGM・SE・ボイスチャンネル分離）②BGMクロスフェード・ループ設定③シナリオ連動のSE再生システム④ボイス再生システム（テキスト表示との同期）⑤音量設定の保存・復元。C#実装コード（AudioSource管理・非同期対応）＋AudioMixerグループ構成＋シナリオファイルでの音声コマンド記述例を出力してください。テーマ：[テーマ]」

4. 全エージェントの完了を待ち、結果を受け取る
