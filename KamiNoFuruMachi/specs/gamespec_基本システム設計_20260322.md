## 変数命名規則（統一版）

本仕様書では以下を**唯一の正式規則**とする。

```
スコーププレフィックス:
  g_  → グローバル（全セーブスロット共通・周回引継ぎ）
  l_  → ローカル（セーブスロット単位）
  t_  → テンポラリ（セッション限り・セーブされない）

セマンティックプレフィックス（スコープの後に付ける）:
  flag_    → Bool型イベントフラグ
  aff_     → 好感度カウンター（Int 0〜100）
  karma_   → 業・穢れカウンター（Int 0〜100）
  route_   → ルート確定フラグ（Bool・排他）
  know_    → 知識・情報取得フラグ（Bool）
  dead_    → キャラクター生死（Bool）
  ex_      → 実績・隠し条件（Bool・グローバル）

例:
  l_flag_rite_intro      → 緘葬初見済み（ローカル）
  l_aff_lilith           → リリス好感度（ローカル）
  l_karma_mayoi          → マヨイ化度（ローカル）
  g_route_lilith         → リリスルート確定（グローバル・周回引継ぎ）
  g_ex_true_unlock       → トゥルーエンド解放（グローバル）
  l_kanata_costume_state → カナタの衣装状態 ("default"/"ritual"/"wounded"/"infiltrate")
```

# 1. シナリオ仕様

## 1-1. シナリオ構造（チャプター/シーン分割）

### 設計原則

- **シーン**を最小単位とし、チャプターはシーンの論理的まとまり
- シーンIDは `[CHAPTER_ID]-[SCENE_NO]-[VARIANT]` 形式（例：`C2-03-A`）
- バリアントサフィックス（A/B/C…）で同一シーンの分岐差分を独立管理
- シーン間の依存関係はフラグのみで制御（順序ハードコード排除）
- 各チャプター末尾に「挿入予約スロット」を確保し、後日シーン追加を無停止で実施可能

### チャプター構成

| ID | チャプター名 | 性質 | シーン数（基幹） | 推定文字数 |
|----|-------------|------|----------------|-----------|
| C0 | 序章「遺骸が降る夜」 | 強制・全共通 | 4 | 約8,000字 |
| C1 | 第一章「ナインフォールの声」 | 強制・全共通 | 8 | 約18,000字 |
| C2 | 第二章「緘葬師の仕事」 | 共通＋初期分岐 | 10+α | 約22,000字 |
| C3 | 第三章「マヨイの海」 | ルート分岐開始 | 12+α×3 | 約30,000字 |
| C4 | 第四章「神の名前」 | ルート固定 | 各ルート8〜10 | 約28,000字 |
| C5 | 終章「降誕と終滅」 | ルート収束〜ED | 各ルート6〜8 | 約24,000字 |
| EX | エピローグ群 | ED種別ごと | 10種 | 約12,000字 |

> **総計目安**：基幹シーン約60、全バリアント含む実シーン数 約110〜130

## 1-2. フラグ管理システム（統一命名規則適用済み）

### フラグ型定義

| 型 | 用途 | 値域 |
|---|---|---|
| Bool (`flag_*`) | 事象成立フラグ | true / false |
| Int (`aff_*` `karma_*`) | 加減算カウンター | 0〜100（クランプ必須） |
| Bool (`route_*`) | ルート確定フラグ（排他） | true / false |
| Bool (`know_*`) | 知識取得フラグ | true / false |
| Bool (`ex_*`) | 実績・隠し条件（グローバル） | true / false |
| String (`*_state`) | 状態列挙型 | 定義済み文字列 |

### 主要フラグ一覧

| 変数名 | 型 | 初期値 | 意味 |
|--------|-----|--------|------|
| `l_flag_rite_intro` | Bool | false | 緘葬の初見済み |
| `l_flag_uta_heard` | Bool | false | ウタを「聞いた」経験あり |
| `l_flag_lilith_trust` | Bool | false | リリスを信頼した |
| `l_flag_arm_exposed` | Bool | false | カナタが右腕の秘密を露出した |
| `l_flag_mayoi_witnessed` | Bool | false | マヨイへの変化を目撃した |
| `l_flag_rite_rejected` | Bool | false | 緘葬を意図的に拒否した |
| `l_flag_empire_known` | Bool | false | 帝国の真の目的を知った |
| `l_aff_lilith` | Int | 0 | リリス好感度 |
| `l_aff_guild` | Int | 0 | 緘葬師ギルドへの忠誠度 |
| `l_karma_mayoi` | Int | 0 | 遺骸汚染・マヨイ化度 |
| `l_karma_empire` | Int | 0 | 帝国への加担度 |
| `l_kanata_erosion_rate` | Float | 0.0 | 神憑き侵食度（0.0〜1.0） |
| `l_kanata_costume_state` | String | "default" | カナタ衣装状態 |
| `g_route_lilith` | Bool | false | リリスルート確定（グローバル） |
| `g_route_empire` | Bool | false | 帝国ルート確定（グローバル） |
| `g_route_mayoi` | Bool | false | マヨイルート確定（グローバル） |
| `g_ex_true_unlock` | Bool | false | トゥルーエンド解放条件達成 |
| `g_ex_clear_lilith` | Bool | false | リリスGOOD END取得済み |
| `g_ex_clear_empire` | Bool | false | 帝国GOOD END取得済み |
| `g_ex_clear_mayoi` | Bool | false | マヨイGOOD END取得済み |

### フラグ汚染防止ルール

1. `g_route_*` フラグは一度trueになると他ルートフラグを強制リセット（排他ロック）
2. カウンター系は上限・下限クランプ必須（0〜100の範囲外に出ない）
3. `g_ex_*` フラグはセーブデータを跨いで引き継ぐ（周回引継ぎ対象）


## 2-3. FlagManager クラス設計

```csharp
public static class FlagManager
{
    private static Dictionary<string, object> _global = new();
    private static Dictionary<string, object> _local  = new();
    private static Dictionary<string, object> _temp   = new();

    public static event Action<string, object> OnFlagChanged;

    public static T Get<T>(string key, T defaultVal = default)
    {
        var store = ResolveStore(key);
        return store.TryGetValue(key, out var v) ? (T)v : defaultVal;
    }

    public static void Set(string key, object value)
    {
        var store = ResolveStore(key);
        store[key] = value;
        OnFlagChanged?.Invoke(key, value);
        if (key.StartsWith("g_")) MarkGlobalDirty();
    }

    private static Dictionary<string, object> ResolveStore(string key) =>
        key.StartsWith("g_") ? _global :
        key.StartsWith("t_") ? _temp   : _local;

    // ルートフラグ排他制御
    public static void SetRoute(string routeKey)
    {
        foreach (var key in new[] { "g_route_lilith", "g_route_empire", "g_route_mayoi" })
            _global[key] = false;
        Set(routeKey, true);
    }
}
```

## 2-6. コアシステム仕様

### 既読スキップ

```
モード1（デフォルト）: 既読のみスキップ
  各セリフにhash（chapter+scene+index）を付与
  ReadLogManager がグローバルに管理
  未読行で自動停止

モード2（設定で解放）: 全文スキップ
  選択肢・イベント分岐点で必ず一時停止

スキップ速度: 1行あたり30ms（15ms / 60ms設定可）
スキップ中: BGMは通常再生・ボイスはカット
```

### オートモード

```
待機時間: ボイス終了後 + [設定値 0.5〜3.0秒]
ボイスなし行: テキスト文字数 × 0.06秒 + 設定値
中断条件: 画面クリック / 選択肢出現
```

### バックログ

```
保持件数: 直近200行（メモリ上）/ セーブ時50行永続化
ボイス再生: バックログ中に個別再生可
```
